PREGUNTAS CONCEPTUALES

Por qué hashear contraseñas?

Para proteger la información sensible de los usuarios en caso de que la base de datos sea comprometida. Un hash es irreversible, lo que significa que no se puede obtener la contraseña original. Bcrypt además utiliza salt y es resistente a ataques de fuerza bruta y rainbow tables.

Ventajas de usar SQLite

  • Base de datos ligera y portátil.
  • No necesita instalación de un servidor adicional.
  • Ideal para proyectos pequeños y prototipos.
  • Fácil de compartir, solo requiere el archivo .db.
